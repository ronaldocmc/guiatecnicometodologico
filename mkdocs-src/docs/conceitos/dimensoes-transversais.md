# Dimensões transversais

As dimensões transversais devem aparecer em todo o ciclo da proposta.

| Dimensão | Pergunta orientadora | Evidência |
|---|---|---|
| Governança | Quem pode acessar, decidir e alterar? | políticas, papéis, matriz de acesso |
| Qualidade | Os dados/artefatos são confiáveis? | relatórios de qualidade, testes |
| Proveniência | Qual a origem e derivação? | grafo de linhagem, manifesto |
| Rastreabilidade | Consigo reconstruir entrada→saída? | matriz ou trilha de auditoria |
| Semântica | O significado está documentado? | glossário, taxonomia, dicionário |
| Fairness | Há desigualdades ou vieses? | métricas por grupo |
| Privacidade | Dados sensíveis foram protegidos? | anonimização, controle de acesso |
| Explicabilidade | Resultado pode ser interpretado? | SHAP, LIME, mapas, justificativas |
| Monitoramento | Mudanças são detectadas? | alertas, drift, observabilidade |
| Reprodutibilidade | O processo pode ser reexecutado? | scripts, ambiente, versões, seeds |
| Catálogo e reuso | Ativos podem ser encontrados e reutilizados? | catálogo, tags, metadados |
