# Tipos de proposta

A primeira decisão do projeto é identificar a natureza da contribuição. Muitos trabalhos usam indevidamente os termos modelo, método, metodologia, arquitetura, framework, plataforma, pipeline e artefato como sinônimos. Este guia diferencia cada tipo.

| Tipo | Pergunta central | Produto mínimo |
|---|---|---|
| Modelo | O que representa? | Diagrama, matriz, taxonomia ou especificação |
| Modelo de referência | Como orientar uma classe de soluções? | Princípios, elementos, relações e instanciação |
| Método | Como executar uma tarefa específica? | Procedimento, implementação, métricas e baseline |
| Técnica | Como aplicar um procedimento operacional? | Antes/depois, parâmetros e impacto |
| Metodologia | Como conduzir um processo? | Etapas, instrumentos, artefatos e aplicação |
| Arquitetura | Como organizar componentes? | Visões, módulos, fluxos e requisitos |
| Framework | Como orientar uma classe de problemas? | Módulos, processos, artefatos e checklist |
| Framework técnico-metodológico | Como integrar técnica e método de aplicação? | Modelo de referência, arquitetura, módulos, templates e validação |
| Plataforma | Como oferecer funcionalidades a usuários? | Interface, dados, funcionalidades e testes |
| Pipeline | Como transformar entradas em saídas? | Execução ponta a ponta, logs e artefatos |
| Agente inteligente | Como executar tarefa com autonomia controlada? | Tarefa, prompts/regras, logs, validação e limites |
| Artefato computacional | Qual produto concreto foi criado? | Dataset, dashboard, catálogo, template, API etc. |
